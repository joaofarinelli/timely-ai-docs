A Timely.ai é uma plataforma SaaS para automação de atendimento com IA. Você cria agentes conversacionais que atendem no WhatsApp, Instagram, widget web e outros canais — sem escrever um prompt do zero, sem precisar de infra, sem dor de cabeça.

A plataforma tem dois ângulos: o **produto** (dashboard, configuração visual, CRM, analytics) e a **API REST** (integração, automação, leitura/escrita de dados via código).

Esta documentação cobre os dois.

## Estrutura da API

Todas as requisições apontam para:

```
https://api.timelyai.com.br/v1
```

A autenticação é feita via header em toda chamada:

```http
x-api-key: SUA_CHAVE_AQUI
```

### Formato de resposta

Toda resposta bem-sucedida segue o envelope:

```json
{
  "data": { ... },
  "meta": {
    "page": 1,
    "per_page": 20,
    "total": 134
  }
}
```

O campo `meta` aparece apenas em rotas de listagem com paginação. Rotas que retornam um único recurso respondem direto no `data`.

### Formato de erro

Qualquer erro retorna o mesmo formato, independente do status HTTP:

```json
{
  "error": {
    "code": "AGENT_NOT_FOUND",
    "message": "O agente informado não existe ou não pertence ao seu workspace.",
    "status": 404
  }
}
```

Consulte a lista completa de códigos em [Rate Limits e Erros](/start/rate-limits).

### Paginação

Rotas de listagem aceitam `page` e `per_page` como query params:

```
GET /v1/agents?page=2&per_page=50
```

O padrão é `page=1` e `per_page=20`. O máximo aceito para `per_page` é `100`.

## Base URL

A API da Timely.ai expõe todos os endpoints a partir de:

```
https://api.timelyai.com.br/v1
```

Para isolar testes, crie um **workspace** dedicado (ex.: "Desenvolvimento") e gere uma API key escopada nele. Dados de workspaces não se misturam.
