title: "Nó Agente (Requisição do Agente)"
description: "Exponha um workflow como ferramenta para agentes IA da Timely.ai."
---

<Note>
  O agente só aciona este workflow se ele estiver **publicado**. Rascunhos não são expostos como ferramentas.
</Note>

## Como Funciona

O trigger **Requisição do Agente** transforma um workflow em uma ferramenta que agentes IA configurados na Timely podem acionar durante uma conversa. Quando o agente identifica que precisa executar o workflow, monta os parâmetros definidos no nó e dispara a chamada.

- Os parâmetros chegam via `{{ $trigger.parameters.nome_do_param }}`
- O workflow consulta APIs externas, processa dados e pode retornar resposta estruturada ao agente
- O modo `node` mantém o agente suspenso até o fluxo atingir um nó Retornar Resposta

## Opções de Configuração

- **Parâmetros** — campos que o agente enviará; cada um tem nome, tipo (`string`, `number` ou `boolean`), descrição e flag de obrigatoriedade
- **Modo de resposta** — `immediate` (agente não aguarda o resultado) ou `node` (agente fica suspenso até o nó Retornar Resposta)

## Casos de Uso Comuns

- **Consulta em tempo real** — agente captura número do pedido na conversa, workflow busca status na API do ERP e retorna o resultado
- **Enriquecimento de contato** — agente coleta e-mail do usuário, workflow consulta API de enriquecimento e salva dados no CRM
- **Classificação e roteamento** — agente passa última mensagem para workflow com nó LLM que classifica a intenção e retorna a categoria
