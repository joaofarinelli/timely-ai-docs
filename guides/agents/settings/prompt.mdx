O prompt de sistema é a instrução mais importante de um agente. Ele define quem o agente é, como ele se comunica, o que pode e o que não pode fazer. O editor de prompt da Timely.ai vai além de um campo de texto simples — ele tem histórico de versões, melhoria automática por IA e análise de qualidade em tempo real.

## Visão geral

O editor fica na aba **Prompt** das configurações do agente. Você escreve o prompt no painel principal e as ferramentas de IA ficam disponíveis na barra lateral.

## Estrutura recomendada de prompt

Um bom prompt de agente tem quatro seções principais:

```
# Identidade
Você é [Nome], assistente de [empresa/área].
Seu objetivo é [objetivo principal].

# Tom e estilo
- [Bullet com regras de comunicação]
- Exemplos: "seja direto", "não use gírias", "use bullet points quando listar itens"

# O que você pode fazer
- [Lista de capacidades permitidas]

# O que você NÃO deve fazer
- [Lista de restrições claras]
- Exemplo: "Nunca mencione concorrentes"
- Exemplo: "Nunca invente preços — consulte sempre a ferramenta X"
```

<Tip>
Inclua exemplos de respostas ideais dentro do prompt quando precisar de um estilo de comunicação muito específico. O modelo aprende por analogia.
</Tip>

## Análise de qualidade em tempo real

Conforme você escreve, o editor analisa o prompt e exibe um painel de sugestões priorizadas. A análise verifica:

- Clareza da identidade do agente
- Presença de restrições explícitas
- Completude das instruções de comportamento
- Possíveis ambiguidades que podem causar alucinações

As sugestões são ordenadas por impacto. Aplique-as clicando no botão ao lado de cada item.

## Melhoria automática por IA

O botão **Melhorar com IA** envia o prompt atual para um modelo especializado que reescreve e otimiza o texto mantendo a intenção original.

<Steps>
  <Step title="Abra o painel de melhoria">
    Clique em **Melhorar com IA** na barra de ferramentas do editor.
  </Step>
  <Step title="Aguarde a sugestão">
    O modelo analisa o prompt e gera uma versão melhorada. O texto sugerido aparece em um painel de comparação.
  </Step>
  <Step title="Aceite ou descarte">
    Clique em **Aceitar** para substituir o prompt atual ou em **Descartar** para manter o original.
  </Step>
</Steps>

<Note>
A melhoria automática preserva as regras de negócio e restrições que você escreveu. Se o resultado não refletir suas intenções, adicione mais contexto ao prompt e tente novamente.
</Note>

## Histórico de versões

Toda vez que você salva o prompt, uma nova versão é criada automaticamente. O histórico fica acessível pelo botão **Histórico** no editor.

<Frame caption="Dialog de histórico de versões do prompt">
  <img src="/images/screenshots/placeholder-agent-prompt-history.png" alt="Histórico de versões do prompt" />
</Frame>

No histórico você pode:

- Visualizar qualquer versão anterior completa
- Comparar a versão selecionada com a atual
- Restaurar uma versão anterior com um clique

<Warning>
Restaurar uma versão anterior substitui o prompt ativo imediatamente. Essa ação não pode ser desfeita — mas o prompt atual é salvo como uma nova versão antes da restauração, então nenhum texto é perdido.
</Warning>

## Limite de caracteres

O prompt tem um limite máximo definido pela constante `PROMPT_MAX_LENGTH` da plataforma. O contador de caracteres fica visível no canto inferior do editor. Prompts muito longos aumentam o custo de tokens por execução.

<Tip>
Se precisar de muita informação contextual, prefira adicionar documentos na aba **Treinamento** em vez de colocar tudo no prompt. O sistema RAG injeta apenas os trechos relevantes para cada mensagem, economizando tokens.
</Tip>

## Prompts condicionais

Você pode usar blocos condicionais no prompt para adaptar o comportamento do agente conforme variáveis de contexto, como o canal de atendimento ou informações do contato.

Exemplo de prompt condicional por canal:

```
{{#if channel == "whatsapp"}}
Use mensagens curtas, máximo 3 linhas por bloco.
Prefira bullet points a parágrafos longos.
{{else}}
Você pode escrever respostas mais detalhadas no widget.
{{/if}}
```

<Info>
As variáveis disponíveis para condicionais variam conforme o canal e o plano. Consulte a [referência de variáveis de prompt](/api-reference/introduction) para a lista completa.
</Info>
