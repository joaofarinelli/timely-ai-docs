## Passo a passo

Conectar um servidor MCP ao agente envolve registrar o servidor, estabelecer a conexão, sincronizar as ferramentas disponíveis e ativar individualmente as que o agente pode usar.

<Steps>
  <Step title="Abra a seção MCP do agente">
    No painel da Timely.ai, abra o agente desejado e acesse a aba de configurações. Selecione **MCP** no menu lateral. Você verá a lista de servidores já conectados ao agente. Clique em **Nova conexão MCP** para iniciar.
  </Step>
  <Step title="Informe os dados do servidor">
    No formulário de nova conexão, preencha:

    - **Nome do servidor** — um rótulo descritivo para identificar o servidor (ex.: "API de Estoque Interno").
    - **URL** — o endpoint do servidor MCP. Pode ser SSE (`/sse`) ou HTTP.
    - **Autenticação** — se o servidor exigir, selecione o tipo (Bearer Token ou API Key) e informe o valor. A credencial é armazenada de forma segura e nunca fica visível na interface após salvar.

    Clique em **Adicionar** para registrar o servidor.
  </Step>
  <Step title="Conecte ao servidor">
    Após registrar, clique em **Conectar** no card do servidor. A Timely.ai abre uma sessão com o servidor MCP e verifica se a comunicação está funcionando. O status muda para "Conectado" quando bem-sucedido — ou exibe uma mensagem de erro indicando a causa da falha (URL incorreta, servidor fora do ar, credencial inválida).
  </Step>
  <Step title="Sincronize as ferramentas disponíveis">
    Com o servidor conectado, clique em **Sincronizar tools**. A Timely.ai consulta o servidor e importa a lista de ferramentas disponíveis. Cada ferramenta aparece com seu nome (slug), a descrição fornecida pelo servidor e os parâmetros aceitos.

    <Note>
    Repita a sincronização sempre que o servidor MCP for atualizado com novas ferramentas ou quando as descrições e parâmetros forem alterados. A Timely.ai não sincroniza automaticamente — é necessário acionar manualmente.
    </Note>
  </Step>
  <Step title="Ative as ferramentas desejadas">
    Por padrão, todas as ferramentas importadas ficam desativadas. Ative individualmente as que o agente pode usar. Para cada ferramenta, você pode editar a descrição exibida ao modelo e adicionar uma instrução de uso — um texto injetado no prompt do sistema que orienta o agente sobre quando e como acionar a ferramenta.
  </Step>
  <Step title="Salve e teste no Playground">
    Salve as configurações e abra o Playground do agente. Faça perguntas ou simule cenários que devem acionar as ferramentas ativadas. Verifique se o agente chama as ferramentas corretas e se o resultado é incorporado adequadamente à resposta.
  </Step>
</Steps>

## Disponibilizar ferramentas MCP para um agente

Se você já tem um servidor MCP registrado no workspace e quer conectá-lo a um agente diferente, o processo é simplificado:

<Steps>
  <Step title="Abra o agente de destino">
    Acesse o agente desejado e vá até a seção **MCP** nas configurações. Clique em **Nova conexão MCP**.
  </Step>
  <Step title="Selecione o servidor existente">
    Escolha o servidor MCP já registrado no workspace na lista de servidores disponíveis. As ferramentas previamente sincronizadas ficam disponíveis imediatamente — não é necessário sincronizar novamente, a menos que o servidor tenha sido atualizado.
  </Step>
  <Step title="Ative as ferramentas para este agente">
    Ative individualmente as ferramentas que fazem sentido para o contexto deste agente específico. Um mesmo servidor pode ter ferramentas ativadas de forma diferente em agentes distintos — cada agente controla quais ferramentas enxerga.
  </Step>
</Steps>

## Teste interno (recomendado)

Antes de colocar o agente com ferramentas MCP em produção, realize testes internos para validar o comportamento:

- Use o Playground com cenários reais do seu fluxo de atendimento — inclua casos em que a ferramenta deve ser acionada e casos em que não deve, para verificar se o modelo distingue corretamente.
- Confirme que os resultados retornados pelo servidor são interpretados e formatados corretamente pelo agente antes de expor o canal aos usuários finais.

## Solução de problemas

Se a conexão ou as ferramentas não funcionarem como esperado, verifique os pontos mais comuns:

- **Falha na conexão** — confirme que o servidor está em operação, a URL está correta e a credencial de autenticação é válida e não expirou.
- **Nenhuma ferramenta aparece após sincronizar** — o servidor pode estar retornando uma lista vazia; verifique a implementação do servidor e confirme que ele expõe ao menos uma ferramenta com schema válido.
